# LocalHub 프로젝트 로컬 데이터 스키마 명세서 (Local Storage Schemas)

본 문서는 백엔드 서버와 데이터베이스가 없는 단일 프론트엔드(Vue 3) 개발 환경에서 `localStorage`를 효율적으로 활용하기 위해 설계된 데이터 형식 정의서입니다. 

기획자(PM)와 프론트엔드 개발자(이진효, 오수빈) 간의 원활한 협업과 데이터 일관성을 위해, **필수 데이터 구조 5종**과 **확장 옵션 데이터 구조 5종**의 상세 스펙(TypeScript Interface 및 JSON Template)을 정의합니다.

---

## 📌 설계 원칙 및 권장사항
1. **키(Key) 관리의 상수화:** 오타로 인한 버그를 방지하기 위해 로컬스토리지 키는 프로젝트 내부 상수 파일(예: `constants/storage.js`)로 정의하여 사용하세요.
2. **독립된 방(Key) 분리:** 모든 데이터를 하나의 덩어리로 묶지 말고, 기능 단위로 개별 키를 적용하여 데이터 조회/수정 속도를 최적화하고 기능 간 결합도를 낮춥니다.
3. **직렬화/역직렬화(Serialization) 유틸리티 사용:** `localStorage`는 문자열만 저장할 수 있으므로, 아래와 같은 래퍼 함수를 미리 만들어두면 편리합니다.

## 📂 PART 1. 필수 데이터 형식 (Must-Have 5종)

프로젝트 핵심 MVP 기능을 구현하는 데 반드시 필요한 데이터 스키마입니다.

### 1. 게시글 메인 데이터

* **Storage Key:** `localhub_posts`
* **설명:** 사용자가 작성한 글 본문과 원본 TourAPI에서 선택한 장소의 핵심 정보(`placeInfo`)를 연결하여 보관하는 배열 형태의 메인 데이터입니다.
* **TypeScript Interface:**

```typescript
interface Post {
  id: number;                // 고유 식별자 (글 작성 시점의 Timestamp)
  title: string;             // 게시글 제목
  content: string;           // 게시글 본문 내용
  password: string;          // 수정/삭제 권한 확인을 위한 비밀번호 (평문)
  placeInfo: {               // 선택한 장소의 핵심 정보 (원본 API 데이터 기반)
    contentid: string;       // TourAPI 공공데이터 고유 식별 ID
    title: string;           // 장소명
    region: string;          // 서울시 자치구 단위 이름 (예: "노원구")
    category: string;        // 카테고리 한글명 (예: "관광지", "문화시설")
  };
  views: number;             // 조회수 (상세조회 시 1씩 증가, 기본값: 0)
  likes: number;             // 좋아요 추천수 (기본값: 0)
  createdAt: number;         // 최초 작성 시각 Timestamp (Unix Time)
}

```

* **JSON Template (초기 Dummy Data):**

```json
[
  {
    "id": 1721003400000,
    "title": "가족들과 주말 나들이로 다녀온 노원기차마을 후기",
    "content": "미니어처 기차들이 정교하게 작동해서 아이들이 정말 좋아합니다. 주차 공간은 다소 협소하니 대중교통 이용을 추천드려요.",
    "password": "1234",
    "placeInfo": {
      "contentid": "4077758",
      "title": "노원기차마을",
      "region": "노원구",
      "category": "문화시설"
    },
    "views": 42,
    "likes": 5,
    "createdAt": 1721003400000
  }
]

```

### 2. 장소 검색용 경량화 데이터

* **Storage Key:** `localhub_places_lite` (또는 앱 구동 시 공공 JSON 원본을 파싱하여 메모리에 업로드)
* **설명:** 원본 공공데이터 파일들(`서울_관광지.json` 등)에서 검색 기능과 카카오/Leaflet 지도 마커 렌더링에 불필요한 필드를 걷어내고 필수 메타데이터만 압축한 구조입니다.
* **TypeScript Interface:**

```typescript
interface PlaceLite {
  contentid: string;   // TourAPI 원본 고유 ID
  title: string;       // 장소명
  address: string;     // 도로명 혹은 지번 주소 (원본의 addr1 필드 활용)
  category: string;    // 관광지, 문화시설, 쇼핑, 레포츠 등 매핑된 카테고리
  mapx: string;        // 경도 (WGS84 좌표계)
  mapy: string;        // 위도 (WGS84 좌표계)
}

```

* **JSON Template:**

```json
[
  {
    "contentid": "4077758",
    "title": "노원기차마을",
    "address": "서울특별시 노원구 화랑로 622 (공릉동)",
    "category": "문화시설",
    "mapx": "127.0941438224",
    "mapy": "37.6252992301"
  },
  {
    "contentid": "1059877",
    "title": "양화한강공원",
    "address": "서울특별시 영등포구 노들로 221 (당산동)",
    "category": "관광지",
    "mapx": "126.9023658810",
    "mapy": "37.5382819489"
  }
]

```

### 3. 챗봇 대화 히스토리

* **Storage Key:** `localhub_chat_history`
* **설명:** 새로고침 시에도 이전 대화 내용을 유지하기 위한 배열입니다. OpenAI Chat Completion API 요청 시 보낼 `messages` 파라미터 구조와 1:1 대응하도록 설계되었습니다.
* **TypeScript Interface:**

```typescript
interface ChatMessage {
  role: "system" | "user" | "assistant";  // 시스템 설정 / 사용자 질문 / AI 답변 구분
  content: string;                        // 메시지 본문 텍스트
}

```

* **JSON Template (초기 세팅 값):**

```json
[
  {
    "role": "system",
    "content": "너는 서울 지역의 나들이 장소를 추천해주는 'LocalHub'의 친절하고 상냥한 AI 가이드야. 한국어로 답변해줘."
  },
  {
    "role": "user",
    "content": "노원구 근처에 갈 만한 문화시설이 있을까?"
  },
  {
    "role": "assistant",
    "content": "네! 노원구 공릉동에 위치한 **'노원기차마을'**을 추천해 드려요. 정교한 디오라마와 미니어처 기차가 가득해서 가족, 친구분들과 특별한 추억을 만들기 아주 좋은 곳이랍니다!"
  }
]

```

### 4. 좋아요 중복 방지 기록

* **Storage Key:** `localhub_user_likes`
* **설명:** 한 사용자가 동일 글에 무한정 '좋아요'를 연타하는 행위를 막기 위해, 현재 브라우저 환경에서 추천을 누른 게시글의 `id` 목록만 담아두는 기록장입니다.
* **TypeScript Interface:**

```typescript
type UserLikes = number[]; // 좋아요를 누른 Post ID(Timestamp)들의 배열

```

* **JSON Template:**

```json
[
  1721003400000,
  1721008800000
]

```

### 5. 대시보드 시각화 통계 데이터

* **Storage Key:** `localhub_dashboard_stats`
* **설명:** Chart.js 렌더링 속도 향상을 위해 매번 수많은 게시글을 배열 탐색하지 않고, 등록/삭제 이벤트가 발생할 때마다 캐싱된 누적 집계 데이터를 업데이트하여 바로 컴포넌트에 주입할 수 있도록 설계한 객체입니다.
* **TypeScript Interface:**

```typescript
interface DashboardStats {
  lastUpdated: number;                  // 최종 업데이트 시간 (Timestamp)
  byRegion: Record<string, number>;     // 각 지역(구)별 총 게시글 개수
  byCategory: Record<string, number>;   // 각 카테고리별 총 게시글 개수
}

```

* **JSON Template:**

```json
{
  "lastUpdated": 1721005000000,
  "byRegion": {
    "노원구": 12,
    "영등포구": 45,
    "종로구": 8,
    "강남구": 3
  },
  "byCategory": {
    "관광지": 25,
    "문화시설": 15,
    "레포츠": 8,
    "쇼핑": 20
  }
}

```

## 📂 PART 2. 확장/선택 데이터 형식 (Optional 5종)

프로젝트 마일스톤에 여유가 있거나, 서비스 고도화 및 보너스 점수를 목표로 할 때 선택적으로 도입할 수 있는 유용한 데이터 구조입니다.

### 6. 사용자 환경 설정 데이터

* **Storage Key:** `localhub_settings`
* **설명:** 유저의 테마 설정 상태 및 기본 선호 지역 정보를 기억하고 앱 진입 시 자동 반영합니다.
* **TypeScript Interface:**

```typescript
interface UserSettings {
  isDarkMode: boolean;            // 다크 모드 활성화 여부
  defaultRegion: string;          // 메인 화면 진입 시 초기 필터링할 구 이름 (기본값: "전체")
  viewMode: "grid" | "list";      // 게시판 목록 형태 렌더링 스타일 (카드형 격자 vs 리스트형)
}

```

* **JSON Template:**

```json
{
  "isDarkMode": false,
  "defaultRegion": "노원구",
  "viewMode": "grid"
}

```

* **Storage Key:** `localhub_bookmarks`
* **설명:** 게시글을 작성하지 않더라도 유저가 관심 있는 장소를 저장해두고 마이페이지나 찜한 장소 탭에서 즉시 확인할 수 있는 기능입니다.
* **TypeScript Interface:**

```typescript
type Bookmarks = string[]; // 장소 찜하기 목록: contentid(string)들의 배열

```

* **JSON Template:**

```json
[
  "4077758",
  "1059877"
]

```

### 8. 최근 검색어 목록

* **Storage Key:** `localhub_recent_searches`
* **설명:** 장소 검색 모달창 진입 시, 유저가 이전에 검색했던 히스토리를 제공하여 검색 피로도를 줄여줍니다.
* **TypeScript Interface:**

```typescript
type RecentSearches = string[]; // 최근 입력한 키워드 문자열 배열 (최대 10개 유지 권장)

```

* **JSON Template:**

```json
[
  "기차마을",
  "한강 돗자리",
  "경복궁 야간개장"
]

```

### 9. 글 작성 임시 저장 데이터

* **Storage Key:** `localhub_post_draft`
* **설명:** 사용자가 글을 작성하다 뒤로 가기, 탭 닫기 등을 수행하여 미완성 글이 유실되는 불상사를 방지하고 앱 재접속 시 복구 기능을 제안할 수 있습니다.
* **TypeScript Interface:**

```typescript
interface PostDraft {
  title: string;             // 입력 중이던 제목
  content: string;           // 입력 중이던 본문 내용
  placeInfo: {               // 연동 중이던 장소 객체 (미선택 시 null)
    contentid: string;
    title: string;
  } | null;
  lastSavedAt: number;       // 자동 저장된 마지막 시간 (Timestamp)
}

```

* **JSON Template:**

```json
{
  "title": "이번 주말 꿀팁",
  "content": "양화한강공원에 텐트 치실 때 주의할 점은...",
  "placeInfo": {
    "contentid": "1059877",
    "title": "양화한강공원"
  },
  "lastSavedAt": 1721004500000
}

```

### 10. 게시글 댓글 데이터

* **Storage Key:** `localhub_comments`
* **설명:** 각 게시물 본문 아래에 댓글을 독립적으로 연결하는 데이터 배열입니다. 게시글 데이터 구조에 하위 배열로 직접 중첩시키는 대신, 별도의 스토리지 키에서 RDBMS 외래키 처럼 `postId` 매핑을 수행하여 관리 품질을 높입니다.
* **TypeScript Interface:**

```typescript
interface Comment {
  id: number;                // 댓글 고유 식별 ID
  postId: number;            // 부모 게시글 ID (Post.id 와 일치하는 외래키 매핑)
  content: string;           // 댓글 본문 내용
  password: string;          // 댓글 삭제/수정 시 권한 확인 비밀번호 (평문)
  createdAt: number;         // 작성 시간 (Timestamp)
}

```

* **JSON Template:**

```json
[
  {
    "id": 1721009999999,
    "postId": 1721003400000,
    "content": "정보 감사합니다! 주말 주차 꿀팁이 아주 유익하네요.",
    "password": "0000",
    "createdAt": 1721009999999
  }
]

```
