## 2. 선택기능 아이디어 평가

**평가기준:** 종합점수 = 실현가능성(1~5) × 임팩트(1~5) × 2 (50점 만점)

| 선택기능 | 구현난이도 | 실현가능성 | 임팩트 | 종합점수 | 선정여부 |
|---|:---:|:---:|:---:|:---:|:---:|
| 데이터 시각화 대시보드 (Chart.js) | 2 | 4 | 4 | **32** | ✅ 선정 |
| 커뮤니티 게시판 추가기능(검색/조회수/좋아요) | 2 | 4 | 4 | **32** | ✅ 선정 |
| 지도 시각화 (Leaflet.js/Kakao) | 3 | 3 | 5 | 30 | 보류 |
| 축제 캘린더 (FullCalendar) | 3 | 3 | 4 | 24 | 보류 |
| 실시간 알림 (Polling) | 3 | 3 | 2 | 12 | 제외 |
| 다국어 지원 (i18n) | 3 | 3 | 2 | 12 | 제외 |
| 경로 안내 | 5 | 1 | 5 | 10 | 제외 |
| 날씨 정보 연동 | 3 | 2 | 2 | 8 | 제외 |
| 소셜 공유 연동 | 4 | 2 | 2 | 8 | 제외 |

**선정 결과:** ① 데이터 시각화 대시보드(32점) ② 커뮤니티 게시판 추가기능(32점)
→ 둘 다 신규 외부 API 키 발급 없이 기존 JSON 데이터·기존 CRUD 로직 위에서 확장 가능해, 3일 일정 + 개발도구 제약(VSCode Copilot + OpenAI API만) 조건에서 가장 안전한 조합.

## 3. MVP 정의서

| 항목 | 내용 |
|---|---|
| 선정 권역 | 서울 |
| 핵심 문제 | 서울에는 관광지·문화시설 등 가볼 만한 장소가 많지만, 사용자가 원하는 지역의 장소를 한눈에 찾고 실제 방문 경험을 공유하기 어렵다 |
| 타깃 사용자 | 서울에서 원하는 지역의 장소를 찾는 시민과 관광객 (회원가입 부담 없는 익명 이용자) |
| 성공 지표 | 납기 내 필수기능 100% 구현 + 선택기능 2개(데이터 시각화 대시보드, 게시판 추가기능) 배포 + 배포 URL 정상 동작 |

### Must have (반드시 포함)

1. 제공 JSON 파일을 프론트엔드에서 직접 연동 (공공 API 직접 호출 불필요, 별도 백엔드 서버 없음)
2. 추가 데이터 수집 시 라이선스·공공누리 유형(1~4) 사전 확인 및 상업적 활용 가능 여부 검토
3. 익명 커뮤니티 CRUD (회원가입/로그인 없음, 인증·권한 체계 미적용) — 목록/상세/작성/수정/삭제
4. 게시글은 localStorage 저장, 제목·내용·수정용 비밀번호 저장, 평문 비교(교육 목적의 의도된 설계), 다른 사용자와 미공유
5. 챗봇 — Vue3에서 OpenAI API 직접 호출, .env(VITE_ 접두사)로 키 관리, 사용량 제한이 걸린 키만 사용 + 결제 한도 낮게 설정 필수
6. 챗봇 주요 질의 유형 응답 — 관광지 추천, 맛집·쇼핑 위치, 게시글 검색 등
7. 채팅 UI — 대화 히스토리 유지, 모바일 대응, 플로팅 버튼
8. FE(화면): Vue.js 3(Vite) 기반 SPA 단일 구조 / 별도 BE(서버) 없음
9. Netlify로 배포 및 URL 정상 동작 확인
10. 기능명세서(활용 데이터 출처·라이선스·수집일 포함) 및 WBS 문서
11. 개발환경 준수 — VSCode Copilot + OpenAI API Key 조합만 사용
12. Claude Code, Codex, Cursor, Antigravity 등 타 AI 코딩 도구 사용 금지 (위반 시 계약 조건에 따라 처리)

### Should / Could have (선정 기능)

- 데이터 시각화 대시보드 — Chart.js로 자치구별 게시글 현황, 장소 유형별(관광지/문화시설/레포츠/쇼핑) 통계 시각화
- 커뮤니티 게시판 추가기능 — 게시글 검색, 조회수 표시, 좋아요 기능
- 관광지·문화시설·레포츠·쇼핑 장소 목록 + 자치구/장소 유형별 필터

### Won't have (이번 MVP 범위에서 제외)

- 회원가입/로그인, 소셜 인증 등 사용자 계정 체계
- 지도 시각화(위치 핀 표시) — 선택기능 슬롯이 이미 채워져 있어 이번 범위 제외
- 서울 외 나머지 4개 권역 데이터 및 기능
- 다국어 지원, 소셜 공유 연동, 실시간 알림, 경로 안내, 날씨 정보 연동, 축제 캘린더

## 5. 데이터 스키마 참고 (개발 시 바로 확인용)

### 파일 최상위 구조

| 필드 | 타입 | 설명 |
|---|---|---|
| `region` | string | 수집 지역명 |
| `contentType` | string | 콘텐츠 유형 한국어명 |
| `contentTypeId` | string | 콘텐츠 유형 ID |
| `total` | integer | 전체 항목 수 |
| `items` | object[] | POI 항목 배열 |

| 25 | 여행코스 | ❌ |
| 28 | 레포츠 | ✅ |
| 32 | 숙박 | ❌ |
| 38 | 쇼핑 | ✅ |
| 39 | 음식점 | ❌ (실제 파일 미보유) |

### items[] 주요 필드

| 필드 | 타입 | 설명 | 개발 시 주의 |
|---|---|---|---|
| `contentid` | string | 콘텐츠 고유 ID | 게시글-장소 연결 키로 활용 가능 |
| `title` | string | 장소명 | - |
| `addr1` / `addr2` | string | 주소 / 상세주소 | 빈 문자열 가능 → 미제공 처리 필요 |
| `mapx` / `mapy` | string | 경도 / 위도 (WGS84) | **string 타입**, float 변환 필요. 이번 MVP는 지도 미사용이라 대시보드 집계용으로만 활용 가능 |
| `cat1~3`, `lclsSystm1~3` | string | 분류 코드 | 코드북(`lclsSystemCode.json`) 없음 → **`contentTypeId` 대분류만 사용** |
| `firstimage` / `firstimage2` | string | 대표 이미지 URL / 썸네일 | 빈 문자열 시 이미지 없음 → placeholder 처리 필요 |
| `createdtime` / `modifiedtime` | string | 등록/수정 시각 (YYYYMMDDHHmmss) | - |
